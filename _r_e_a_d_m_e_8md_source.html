4 ## Introduction
5 The QPathEdit class was designed to provide a widget that allows the user to easily enter or select a local file or directory path. The widget comes with a plugin for QtCreator.
6 
7 ## Details
8 ### The QPathEdit Class
9 The QPathEdit class provides the following features:
10  - **Appereance:** The Edit is composed of a QLineEdit and QToolButton. It uses natives styles. It should look "normal" on any plattform. It provides 3 different styles to allows customisation of the way these components are displayed.
11  - **Dialog:** The user will be able to select his path using QFileDialog. The dialog can be customized to a limited amount.
12  - **Validation:** The edit automatically validates entered and selected paths and makes shure, that it always returns a valid path to your program.
13  - **Editable:** The edit can be editable or readonly. If editable, the validator will mark invalid paths and an auto completer will simplify path entering.
14  - **Completer:** The edit uses an auto completer, that uses the underlying file-system to provide paths as autocompletion to make entering paths even easiert. It can be turned on and off.
15  - **QtCreator Plugin:** The edit widget comes with an designer-plugin for Qt-Creator, to make it possible to user the pathEdit insider .ui files.

19 ### Usage - A simple Example
20 Since the QPathEdit is just a simple widget, you can use it like any other widget.
21 The following example will create a simple window with the QPathEdit inside. If the user enters a path, the new path will be logged on the console:
22 
23 ```cpp
24 #include <QApplication>
25 #include <QWidget>
26 #include <QVBoxLayout>
27 #include <QPathEdit>
28 
29 int main(int argc, char *argv[])
30 {
31  QApplication a(argc, argv);
32 
33  QWidget window;
34  QVBoxLayout *layout = new QVBoxLayout(&window);
35  QPathEdit *edit = new QPathEdit(&window);
36  layout->addWidget(edit);
37  window.setLayout(layout);
38 
39  QObject::connect(edit, &QPathEdit::pathChanged, [](QString path){
40  qDebug() << path;
41  });
42 
43  window.show();
44  return a.exec();
45 }
46 ```

48 ### Installing the Plugin
49 To install the plugin, you need to copy the right file from the `designerplugins.zip` zip-package to the QtCreators designer plugin path. There are a number of subfolders for operating systems I've created the plugin for. If yours is not present, you need to compile the plugin yourself. Copy file (for example `qpatheditplugin.dll`) into QtCreators path. The default path would be:
50 - Windows: `<path_to_qt>/Tools/QtCreator/bin/plugins/designer`
51 - OsX: `<qt_creator_bundle>/Contents/PlugIns/designer`
52 - Linux: `<path_to_qt>/Tools/QtCreator/lib/Qt/plugins/designer`
53 
54 After restarting the creator, navigate to the designer and to "Tools > Form Editor > About Qt Designer Plugins". The plugin should appear there. In the editor itself, you can find it inside the "Input Widgets" Section.
55 
56 For more details, check [Adding Qt Designer Plugins](http://doc.qt.io/qtcreator/adding-plugins.html).
